# Data Infrastructure

Data pipelines/ELT, analytics and transactional databases, data integration unified APIs, and computer-vision data tooling. This is the layer agents stand on: every agent product eventually hits the same wall. It needs trustworthy, queryable, governed access to enterprise data, somewhere durable to keep its own state and memory, and telemetry storage for the traces it emits. The companies here solved those problems for humans and web apps first, and nearly all of them spent 2025–26 re-pitching the same engine at agents: MCP servers are now table stakes (Airbyte, ClickHouse, PlanetScale, YugabyteDB, dbt, Merge, PromptQL all ship one), and several have gone further with purpose-built agent-context or agent-memory products.

## How to read this category

Three entries carry the most weight. **[ClickHouse](clickhouse.md)** is the scale anchor: $250M ARR, a ~$15B valuation, and, via the Langfuse acquisition, ownership of both the storage engine and the most-used open-source LLM tracing tool for agent observability. **[dbt Labs](dbt-labs.md)** (now merged with Fivetran) is the "governed context" bet, with the Semantic Layer plus MCP server as the standard interface between agents and the warehouse. **[Airbyte](airbyte.md)** is the cleanest example of the category's pivot pattern: classic ELT repositioned as a pre-indexed "Context Store" so agents make fewer, cheaper tool calls.

The rest differentiate along two axes:

- **Where they sit relative to the agent.** *Under* it as state/memory stores: the OLTP databases [PlanetScale](planetscale.md), [YugabyteDB](yugabytedb.md) (with its agent-native Meko launch), and [SurrealDB](surrealdb.md) (multi-model engine betting agent memory belongs in one database, not a Postgres + vector + graph sandwich). *Beside* it as the data/tool access layer: [Merge](merge.md) (unified API + Agent Handler governance), [PromptQL](promptql.md) (natural-language program execution over enterprise data), [Edge & Node](edge-and-node.md) (verifiable on-chain data). *Before* it in the training-data pipeline: the computer-vision pair [Encord](encord.md) (annotation/curation/eval, "physical AI") and [Roboflow](roboflow.md) (CV build-train-deploy, `supervision`), plus [Granica](granica.md) (data-lake compression, PII screening, sample selection).
- **Funding posture.** Roughly half the category hasn't raised since the 2021 vintage (Airbyte, PlanetScale, YugabyteDB, Merge, dbt pre-merger) and their agent pivots read partly as growth-story resets; the other half raised fresh 2025–26 money explicitly on the AI narrative (ClickHouse, SurrealDB, Encord).

## Related papers

The profiles in this category repeatedly cite the same research threads. These are the ones that genuinely map:

- [Are We Ready for an Agent-Native Memory System?](../../papers/memory-research-agents/are-we-ready-for-an-agent-native-memory-system.md): the question YugabyteDB's Meko, SurrealDB 3.0, and PlanetScale's agent-state positioning are all commercial answers to.
- [Memory for Autonomous LLM Agents: Mechanisms, Architectures, and Evaluation](../../papers/memory-research-agents/memory-for-autonomous-llm-agents-mechanisms-architectures.md): the gap between raw vector stores and memory carrying entities, relations, and time; exactly what SurrealDB's context graphs and Airbyte's entity-linked Context Store claim to close.
- [Do Agents Need Semantic Metadata? A Comparative Study in Agentic Data Retrieval](../../papers/memory-research-agents/do-agents-need-semantic-metadata-a-comparative-study-in-age.md): the most load-bearing paper for this category, cited by Airbyte, ClickHouse, PlanetScale, YugabyteDB, SurrealDB, dbt, and Encord as the thesis their metadata/semantic-layer products operationalize.
- [The Evolution of Tool Use in LLM Agents: From Single-Tool Call to Multi-Tool Orchestration](../../papers/tool-use-governance/the-evolution-of-tool-use-in-llm-agents-from-single-tool-ca.md): the "fewer, better tool calls" argument behind Airbyte's Context Store and the telemetry needs ClickHouse/Langfuse serve.
- [Governance by Construction for Generalist Agents](../../papers/tool-use-governance/governance-by-construction-for-generalist-agents.md): PlanetScale's branch/deploy-request gate for agent schema changes, Merge's Agent Handler, and dbt's governed metric surface are all this pattern in production.
- [VoiceAgentRAG: Solving the RAG Latency Bottleneck in Real-Time Voice Agents](../../papers/voice-realtime/voiceagentrag-solving-the-rag-latency-bottleneck-in-real-ti.md): the pre-indexing/latency problem Airbyte attacks generally and SurrealDB's PolyAI voice deployment hits live.
- [Reproducible, Explainable, and Effective Evaluations of Agentic AI for Software Engineering](../../papers/swe-agents/reproducible-explainable-and-effective-evaluations-of-agen.md): Encord's human-review-queue discipline applied to grading agent trajectories, and the eval-run data that lands in ClickHouse-scale stores.
- [SWE-EVO: Benchmarking Coding Agents in Long-Horizon Software Evolution](../../papers/swe-agents/swe-evo-benchmarking-coding-agents-in-long-horizon-software.md) and [SlopCodeBench](../../papers/swe-agents/slopcodebench-benchmarking-how-coding-agents-degrade-over-l.md): the long-horizon degradation risk that makes governed, revertible database/DAG changes (PlanetScale deploy requests, dbt Wizard) more than a convenience.

## Open questions

1. **Does agent memory consolidate into general databases or stay a specialized layer?** SurrealDB, YugabyteDB (Meko), and PlanetScale are betting the database absorbs it; Mem0/Zep-style middleware and the vector-native stores are betting the opposite. Nobody has production evidence yet: Meko and Airbyte's Context Store are both weeks-to-months old with no documented production users.
2. **Does MCP commoditize the connector business?** Merge, Airbyte, and PromptQL all sell curated connector catalogs; if every SaaS ships its own MCP server, the moat moves up-stack to governance, entity resolution, and semantics, which is visibly where all three are racing.
3. **Will warehouse vendors' own agents bypass the semantic layer?** dbt's MCP + Semantic Layer as the default agent-to-warehouse interface competes directly with Snowflake Intelligence and Databricks Genie owning that interface natively.
4. **How much of the 2021-vintage half of this category is pivot-as-refinancing?** Airbyte ($1.5B), PlanetScale, YugabyteDB ($1.3B), and Merge have raised nothing in 4+ years; their agent narratives will be tested by whether fresh capital or revenue follows.
5. **Is "physical AI" data infrastructure a real new market or a rebrand?** Encord's 10x physical-AI revenue claim vs. its annotation core, and Roboflow's edge-inference push, are the two entries to watch for whether robotics/AV data tooling breaks out of the CV-labeling niche.
