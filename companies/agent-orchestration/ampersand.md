# Ampersand

> Declarative platform for building native, bi-directional SaaS-to-SaaS integrations (CRM/ERP/GTM) that AI agents can call as tools.

- **Category:** agent-orchestration
- **AIEWF 2026 tier:** silver
- **Founded:** 2022 — Verified
- **Website / GitHub:** https://www.withampersand.com | https://github.com/amp-labs

## What they do
Ampersand lets product/engineering teams ship customer-facing integrations (Salesforce, HubSpot, NetSuite, Marketo, Dynamics, hundreds more) via a version-controllable, YAML-based configuration instead of bespoke OAuth/sync code per connector. It handles auth, field mapping, scheduled/on-demand sync, bulk writes, and — via its newer **Subscribe Actions** — sub-second webhooks, positioning it as an "actions" layer AI agents can call to read/write real CRM data.

## Founders, funding & evidence of use
Founded by Ayan Barua (ex-VPE at G2, co-founder/CTO of Siftery) and Lauren Long (ex-Firebase Extensions, Google) — **Verified**. Raised $4.7M seed (2023, led by Matrix Partners) — **Verified** (TechCrunch). Named customer 11x (an AI phone-agent company) reports cutting agent response time from 60s to 5s using Ampersand; other named users include Clarify, Centralize, Crunchbase, Databook.

**Note:** distinct from the similarly named "Ampersend" (ampersend.ai) — a separate, unrelated "control layer for the agent economy" company; do not conflate.

## Relevance to agentic AI engineering
Tool-calling/action infrastructure for agents that need real enterprise-system access — adjacent to MCP-style action layers and agent-orchestration frameworks in this repo.
