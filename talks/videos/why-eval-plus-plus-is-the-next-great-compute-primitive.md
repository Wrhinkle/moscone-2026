# Why Eval++ Is the Next Great Compute Primitive

> Cloudflare's agents team makes the case that durable objects plus dynamic workers, safe on-demand execution of LLM-generated code, are the compute primitives agent infrastructure has been missing.

- **Speaker:** Sunil Pai & Matt Carey, Cloudflare
- **Video:** [Watch on YouTube](https://www.youtube.com/watch?v=SKDJo2CopRs) (AI Engineer channel; ~25m, released 2026-06-08)
- **Program:** AI Engineer 2026 release, program placement unconfirmed

## Summary

This is a loose, audience-driven session from two engineers on Cloudflare's agents team, and its through-line is a pair of primitives. The first is durable objects. Pai explains them with a hit counter: a serverless function that increments a local variable always returns zero because the isolate vanishes between requests, but a durable object is a class that spins up once for a given ID, and every future request and websocket connection lands in the same place. Cloudflare calls this stateful serverless. Durable objects are addressable, run long tasks with no inbound requests, hibernate, persist, and ride the network's roughly 15ms latency from London, which is under a 60fps frame budget; TLDraw's multiplayer sync is built on them. Pai argues this is the natural compute unit for agents, and it is what the Cloudflare agents SDK wraps, with scheduling ("every Friday at 9 p.m., compile my Git history and wiki into a report for my manager"), React hooks, and a production backend for Vercel's AI SDK. Durable objects also held the stateful connections that early MCP servers needed, which is why PayPal, Sentry, Linear, and Intercom run their MCP servers there.

The second primitive is new: dynamic workers, which the title's Eval++ refers to. From one worker you take a string of code, sent by a customer or generated by an LLM, and run it in its own isolated worker, billions on demand, without deploying anything. Pai frames it as reopening a branch of the tech tree closed for decades: everyone was taught never to use eval, and dynamic workers are eval plus plus, fast, secure, and cheap. The security model inverts the sandbox tradition. Instead of starting from a VM or container and wrapping security around it, the isolate starts with nothing, no fetch, no APIs, no environment variables, and you grant explicit capabilities from outside, down to allowing outgoing requests only to a specific GitHub path. Carey recounts a startup founder insisting no enterprise would run code an LLM generated, followed by a Lockheed Martin engineer saying they love exactly that. Use cases offered: replace generated-JSON DSLs by just generating and running code, and generative UI that renders LLM-written React rather than JSON schemas. Carey's companion talk covers code mode, exposing all 2,600 Cloudflare API endpoints through an MCP server in only about 1,000 tokens by having the model write code against them.

The teases are concrete. The pair say Cloudflare is building its own coding-agent harness on workers, an agent loop in a durable object that any client, terminal, phone, or web app can attach to with resumable streams and multi-device sync out of the box; Pai calls multiplayer AI a gap OpenAI has not closed even for chat sharing. They describe the ingredients of an OpenClaw-style personal agent, a heartbeat, a virtual file system (shipped as a layer over durable object SQLite and R2), connected services, and extensions running in dynamic workers, and mention a new CMS whose plugin system runs entirely on dynamic workers to dodge the WordPress plugin security problem. JavaScript and Python are first class, other languages arrive via WASM, and Pai closes by noting Cloudflare's free tier is structural rather than a marketing expense, since the company buys bulk bandwidth near ISPs instead of building data centers.
